# MoviePilot-Plugins

本仓库为 [MoviePilot](https://github.com/jxxghp/MoviePilot) 第三方插件库，当前提供 **TMM 元数据转移助手** 插件。

---

## TMM 元数据转移助手 (TMMMover)

整合 **实时硬链接**、**TinyMediaManager (TMM)** 刮削转移与 **MoviePilot 原生风格入库通知** 的一体化媒体整理助手，支持跨挂载点迁移，并在成功入库后发送精美图文通知。

- **作者**：QB  
- **版本**：2.1.3
- **适用**：MoviePilot 插件市场（V2）

### 功能概览

| 能力 | 说明 |
|------|------|
| **电影** | 从来源监控目录扫描已刮削电影，按配置迁移到目标电影目录。 |
| **剧集** | 根据 `tvshow.nfo` 中的国家/类型信息，自动归入目标剧集根目录下的分类子目录。 |
| **剧集分类** | 大陆剧集、动漫、短剧、港台剧集、纪录片、欧美剧集、日韩剧集、综艺；类型（动漫/短剧/纪录片/综艺）优先于地区，未匹配时默认归入「欧美剧集」。 |
| **跨挂载点迁移** | 使用目录移动方式迁移，支持来源与目标在不同磁盘或挂载点。 |
| **TMM 废弃清理** | 自动识别并删除名为 `.deletedByTMM` 或 `*.deletedByTMM` 的废弃目录。 |
| **实时硬链接整合** | **新增核心能力**：将 QB 下载目录 A 中的新文件实时硬链接到 TMM 刮削目录 B，按原相对路径同步，完美替代单独的实时硬链接插件。 |
| **未完成下载保护** | **新增安全能力**：实时硬链接会等待文件静默达到指定秒数后再生成；入库前还会再次检查媒体文件最近是否仍在写入，尽量避免 QB 未完成文件被提前刮削后误入库。 |
| **统一插件日志** | 硬链接链路统一显示为 `实时监控`；迁移链路会按触发来源区分为 `手动转移` 或 `定时转移`，更方便在 MP 插件日志里排查问题。 |
| **精美入库通知** | **核心特性**：彻底抛弃繁杂的任务汇总。提取 NFO 公网图源与纯正数据发送通知；对于连季入库的剧集，智能合并季数（如 S01-S03 或 S01,S03-S05）；自动汉化电影的英文流派。 |
| **极致静默模式** | 遇到未刮削、未规范命名或目标目录已存在的资源时，插件仅在系统后台日志中记录明细，绝不在消息中心打扰用户。 |

### 使用场景

- **TMM 刮削后的无缝入库**：用 TMM 在临时目录刮削好的媒体，插件会自动按类型迁移到媒体库，并推送一条原生级别的“已入库”精美通知，仪式感拉满。
- **QB 下载 + 实时硬链接 + TMM 刮削 + 自动入库**：资源先下载到 A 目录，本插件实时硬链接到 B 目录供 TMM 刮削，刮削完成后再由同一个插件转移入媒体库，一套配置完成全流程。
- **跨磁盘/挂载点整理**：来源和目标可以不在同一块盘，插件通过移动目录完成迁移，适合多盘或外挂盘整理到主媒体盘。
- **定时自动整理**：配合 Cron 定时执行，例如每小时扫一次来源目录，新刮削完成的媒体自动进库并推送消息，全程零人工干预。
- **保持目录整洁**：静默清理 TMM 删除媒体后遗留的标记目录，维持下载盘干净清爽。

### 怎么使用

1. **安装插件** 在 MoviePilot 中进入 **插件** → **插件市场**，搜索「TMM 元数据转移助手」或添加本仓库地址后安装并启用。

2. **配置路径** 打开插件 **设置**：  
   - 若你要走“QB 下载 -> TMM 刮削 -> 自动入库”链路：
     - 填 **电影下载目录 A** 与 **电影 TMM 刮削目录 B**
     - 填 **剧集下载目录 A** 与 **剧集 TMM 刮削目录 B**
   - 再填写 **目标电影存放目录**（如 `/media/library/Movies`）和 **目标剧集存放根目录**（如 `/media/library/Series`）。
   - 电影和剧集都支持分别配置，只填你需要的部分即可。

3. **可选配置**
   - **启用实时硬链接**：开启后会实时监控下载目录 A，并将新文件硬链接到 TMM 刮削目录 B。
   - **保存后全量补链一次**：适合首次接入，把已有下载资源一次性补到 TMM 刮削目录。
   - **下载完成保护秒数**：实时硬链接只有在文件最后修改时间静默达到该秒数后才会生成，推荐保留默认值或按你的下载速度适当调大。
   - **定时执行 TMM 转移 Cron**：默认 `0 * * * *`（每小时执行一次）。
   - **入库前静默保护分钟数**：即便 TMM 已经提前刮削，只要目录里媒体文件最近仍有写入痕迹，TMMMover 也会暂缓入库。
   - **启用精美入库通知**：开启后，每部【成功转移】的影视剧都会发送独立的图文入库通知，不打扰，仅在后台日志记录跳过/失败信息。

4. **保存并启用** 保存配置后确保插件处于 **启用** 状态。

5. **执行迁移** - **手动**：打开插件 **详情页**，点击 **立即运行**，任务在后台执行。
   - **定时**：若已配置 Cron，到点会自动执行。  
   - **API**：可调用 `POST /plugin/TMMMover/run`（需 Bearer 认证）。

### 安全与跳过规则

- **重命名规范**：仅处理目录名中包含年份括号的媒体目录（如 `剧名 (2024)` 或 `剧名（2024）`）。
- **刮削校验**：无 NFO 或剧集缺少 `tvshow.nfo` 视为未刮削完成，直接跳过。
- **下载完成保护**：检测到 `.!qB`、`.part`、`.tmp` 等临时下载文件时，硬链接和入库都会自动跳过；即使没有临时后缀，也会根据文件最近写入时间做静默保护。
- **防覆盖保护**：若目标路径下已存在同名目录，则跳过迁移。
- **原子级通知**：通知逻辑被严格放置在跨盘物理移动（`shutil.move`）成功之后，任何磁盘满或权限异常导致的移动失败都绝不会误发入库通知。
