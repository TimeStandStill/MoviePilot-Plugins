# MoviePilot-Plugins

本仓库为 [MoviePilot](https://github.com/jxxghp/MoviePilot) 第三方插件库，当前提供 **TMM 元数据转移助手** 插件。

---

## TMM 元数据转移助手 (TMMMover)

根据 **TinyMediaManager (TMM)** 的 NFO 元数据，自动分拣媒体目录并支持**跨挂载点迁移**，同时自动清理 TMM 标记的废弃目录。

- **作者**：QB  
- **版本**：1.1.12  
- **适用**：MoviePilot 插件市场（V2）

### 功能概览

| 能力 | 说明 |
|------|------|
| **电影** | 从来源监控目录扫描已刮削电影，按配置迁移到目标电影目录。 |
| **剧集** | 根据 `tvshow.nfo` 中的国家/类型信息，自动归入目标剧集根目录下的分类子目录。 |
| **剧集分类** | 大陆剧集、动漫、短剧、港台剧集、纪录片、欧美剧集、日韩剧集、综艺；类型（动漫/短剧/纪录片/综艺）优先于地区，未匹配时默认归入「欧美剧集」。 |
| **跨挂载点迁移** | 使用目录移动方式迁移，支持来源与目标在不同磁盘或挂载点。 |
| **TMM 废弃目录清理** | 自动识别并删除名为 `.deletedByTMM` 或 `*.deletedByTMM` 的目录。 |
| **执行结果通知** | 可选开启：**仅在有成功转移或发生错误时推送通知，避免零变动打扰**。按「成功/未规范或未刮削/目标已存在/失败」分组通知；成功项全量展示，异常类仅展示前 3 条并自动截断超长名称。 |

### 使用场景

- **TMM 刮削/重命名后的自动归位**：用 TMM 在「下载盘」或临时目录里刮削、重命名好的电影/剧集，希望自动按类型迁移到 NAS 或媒体库目录，无需手动拖拽。
- **下载目录与媒体库分离**：下载或导入统一放在一个来源目录，插件按 NFO 分类迁移到电影目录、剧集根目录下的动漫/大陆剧/欧美剧等子目录，便于 Plex/Jellyfin 等刮削与浏览。
- **跨磁盘/挂载点整理**：来源和目标可以不在同一块盘或同一挂载点，插件通过移动目录完成迁移，适合多盘或外挂盘整理到主媒体盘。
- **定时自动整理**：配合 Cron 定时执行，例如每小时或每天凌晨扫一次来源目录，新刮削完成的媒体自动进库。
- **清理 TMM 删除的目录**：TMM 删除媒体后会留下 `.deletedByTMM` 等目录，插件扫描时会自动删除这些废弃目录，保持来源目录干净。

### 怎么使用

1. **安装插件** 在 MoviePilot 中进入 **插件** → **插件市场**，搜索「TMM 元数据转移助手」或添加本仓库地址后安装并启用。

2. **配置路径** 打开插件 **设置**：  
   - 填 **电影来源监控目录**（如 `/media/download/Movies`）和 **目标电影存放目录**（如 `/media/library/Movies`）。  
   - 填 **剧集来源监控目录**（如 `/media/download/Series`）和 **目标剧集存放根目录**（如 `/media/library/Series`）。  
   电影和剧集可只配其一，只填需要的即可。

3. **可选配置** - **定时执行 Cron**：默认 `0 * * * *`（每小时执行一次）。需要自定义自动整理频率时可修改，例如 `0 2 * * *` 表示每天凌晨 2 点执行。  
   - **启用执行结果通知**：需要收到迁移消息时打开，结果会发到系统消息通道（如 Telegram 等已配置的渠道）。无变动时不打扰。

4. **保存并启用** 保存配置后确保插件处于 **启用** 状态。

5. **执行迁移** - **手动**：打开插件 **详情页**，点击 **立即运行**，任务在后台执行，进度可在 **系统** → **日志** 中查看。  
   - **定时**：若已配置 Cron，到点会自动执行。  
   - **API**：可调用 `POST /plugin/TMMMover/run`（需 Bearer 认证）与其他自动化联动。

> 建议先用少量已刮削、命名规范的目录试跑一次，确认目标路径和分类符合预期后再扩大来源目录或开启定时。

### 配置项

- **电影来源监控目录** / **目标电影存放目录**：电影迁移的源与目标。
- **剧集来源监控目录** / **目标剧集存放根目录**：剧集迁移的源与目标；插件会在目标根目录下自动创建分类子目录（如：动漫、大陆剧集等）。
- **定时执行 Cron**：按 cron 表达式定时扫描并执行迁移。
- **启用执行结果通知**：开启后，发生实质性迁移或错误时会发送通知到系统消息通道。

### 安全与跳过规则

- **重命名规范**：仅处理目录名中包含年份括号的媒体目录（如 `剧名 (2024)` 或 `剧名（2024）`），未规范命名的目录会安全跳过。
- **刮削校验**：无 NFO 的目录视为未刮削完成，直接跳过；剧集必须存在 `tvshow.nfo` 才会参与分类与迁移。
- **不覆盖已有目录**：若目标路径下已存在同名目录，则跳过迁移并记录日志。

### 使用方式

- **插件详情页**：点击「立即运行」可手动触发一次迁移任务（后台执行，进度见系统日志）。
- **API**：`POST /plugin/TMMMover/run`（需 Bearer 认证），用于手动触发或与其他流程联动。
- **定时任务**：配置 Cron 后，由 MoviePilot 定时服务自动执行。

### 最近升级（v1.1.12）

- **通知逻辑优化**：定时运行且无新增转移及无报错时不再发送通知，有效避免频繁打扰。
- **默认定时调整**：默认 Cron 任务调整为每小时执行一次（`0 * * * *`）。
- **界面体验增强**：保留 v1.1.11 的通知文本优化成果（成功迁移全量列出，异常类最多展示 3 条并自动截断超长名称）。
